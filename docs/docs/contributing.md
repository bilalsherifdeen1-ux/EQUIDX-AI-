# Contributing

See [`CONTRIBUTING.md`](https://github.com/equidx-ai/equidx-ai/blob/main/CONTRIBUTING.md)
in the repository root for the full guide, covering:

- Ground rules (synthetic data only, disclaimers, architecture boundaries)
- Local development setup
- How to make a change (branching, tests, linting, PR checklist)
- How to add a new AI diagnostic domain
- Code style
- Reporting security issues

## Project board

We track work with a standard GitHub Projects board with columns:

`Backlog → Ready → In Progress → In Review → Done`

Labels mirror the monorepo layout (`web`, `backend`, `ai-engine`,
`biosensor-simulator`, `analytics`, `mobile-api`, `infrastructure`,
`docs`) plus cross-cutting labels (`bug`, `enhancement`, `security`,
`good-first-issue`).
